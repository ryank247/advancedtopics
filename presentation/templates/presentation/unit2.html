We learned how to use SQL to store information for models and objects, which would later be used to enhance a changing website.

SQL allowed us to retrieve and set items/characteristics to objects such as seen in this image:

By having tables like this, it makes it easier for our website to work, for example, if I needed to find the length of a flight to put on a different page of the website.

Something that we used with SQL are Django Models which offer a multitude of field types:
  • Char - Strings (Ex: names)
  • Int - Integers (Ex: age)
  • Boolean - True / False Boolean Values (Ex: is male or not)
  • Foreignkey - Many to One Relationship (Ex: child to their father)

Now that you have these models, you have to make migrations to create a database to store this information of the models:
To do so:
First, python manage.py makemigrations, which creates files to store our models.
Next, python manage.py migrate, which then applies migrations to the database, whcih just helps your code run correctly and have the data stored.
